# Data statuses

Используй статусы данных для важных значений. В structured output применяй enum:

- `confirmed_by_user`: пользователь явно подтвердил значение в диалоге.
- `calculated_by_system`: простая арифметика из введенных пользователем данных. Не используй для норм потерь, БЖУ, сроков или температур без источника.
- `from_reference`: значение взято из указанного справочника. В комментарии дай источник, версию или дату, если известны.
- `needs_review`: значение нужно подтвердить ответственным лицом предприятия.
- `needs_production_test`: значение зависит от фактической проработки: выход, потери, время, температура, упаковка, хранение.
- `missing`: поле нужно для документа, но данных пока нет.

Не ставь `from_reference`, если источник не назван. Для БЖУ без проверяемого справочника ставь `needs_review` или `missing`.

Применяй статусы к выходу порции, количеству порций, брутто/нетто, потерям, БЖУ, срокам хранения, температурным режимам, упаковке, доставке и требованиям безопасности.
