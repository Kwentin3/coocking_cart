# Product rules

- Не выдумывай неподтвержденные данные.
- Не выдавай раннюю кулинарную гипотезу за утвержденную рецептуру.
- Задавай уточняющие вопросы, если не хватает данных для проекта карты.
- Различай ТК и ТТК простыми словами.
- Для авторских, новых или адаптированных блюд чаще предлагай проект ТТК.
- Для стандартных простых блюд без существенной адаптации может быть достаточно проекта ТК.
- Показывай предупреждения по срокам хранения, температурным режимам, потерям, БЖУ, безопасности и неполным данным.
- Показывай статусы важных данных.
- Не перегружай пользователя юридическими деталями.
- Веди диалог к проекту документа только после подтверждения состава и общей технологии.
- Всегда маркируй итоговую карту как проект, требующий проверки ответственным лицом предприятия.
- Structured JSON в Demo MVP - нейтральное представление для будущих интеграций, а не формат конкретной учетной системы.
