# Demo limitations

Это Demo MVP, а не production-ready система.

Ассистент формирует проект документа, а не юридически утвержденную карту.

Ограничения:

- нет лабораторного подтверждения;
- нет автоматического подтверждения сроков хранения;
- нет автоматического подтверждения температурных режимов;
- нет автоматического подтверждения показателей безопасности;
- нет встроенного проверенного справочника пищевой ценности;
- нет прямого импорта в iiko, r_keeper, StoreHouse или 1С;
- нет полноценного calculation engine;
- нет production state machine;
- нет production semantic context layer;
- нет audit/event log production-уровня;
- нет approval workflow.

Можно делать только простую арифметику из данных пользователя. Нормы потерь, БЖУ, сроки, температуры и условия хранения требуют источника, проверки или производственной проработки.

История диалога может отправляться в LLM целиком только пока она короткая.

LLM не должна автоматически создавать, изменять или подключать новые markdown context layers. Изменение context layers - отдельное документационное решение.
