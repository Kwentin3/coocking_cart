# Product workflow: AI-ассистент технологических карт v0.2

- Дата: 2026-05-20
- Статус: продуктовый refine v0.2
- Область: путь пользователя от идеи блюда до проекта документа

## Цель workflow

Показать простой путь пользователя без технических деталей: как человек приходит с идеей блюда, обсуждает ее с ассистентом и получает проект технологической или технико-технологической карты.

В v0.2 отдельно показано, где данные подтверждает пользователь, где система выводит предупреждения, где появляется граница "проект документа" и где начинается зона ответственности технолога или предприятия.

## Общий путь пользователя

1. Пользователь авторизуется.
2. Открывает чат с ассистентом.
3. Описывает блюдо свободным текстом.
4. Ассистент предлагает кулинарную гипотезу.
5. Пользователь корректирует гипотезу.
6. Пользователь подтверждает состав и общий технологический процесс.
7. Ассистент формирует проект ТК или ТТК.
8. Пользователь видит предупреждения, статусы данных и незаполненные поля.
9. Пользователь передает проект ответственному лицу для проверки, производственной проработки и утверждения.

## Статусы данных

В workflow используются следующие статусы:

- **Подтверждено пользователем** - пользователь явно согласился с данными в диалоге.
- **Рассчитано системой** - значение получено из введенных данных.
- **Взято из справочника** - значение связано с источником справочных данных.
- **Требует проверки** - значение нужно подтвердить до утверждения документа.
- **Требует производственной проработки** - значение зависит от фактического приготовления.
- **Не заполнено** - поле нужно, но данных пока нет.

## Где пользователь подтверждает данные

Пользователь должен явно подтвердить:

- формат заведения;
- тип блюда: стандартное, авторское или адаптированное;
- состав ингредиентов;
- выход порции;
- количество порций;
- общий технологический процесс;
- желаемые ограничения по доступности, редкости и дороговизне ингредиентов;
- готовность сформировать проект ТК или ТТК.

Подтверждение пользователем не равно юридическому утверждению документа. Это только переход от обсуждения блюда к проекту документа.

## Где система показывает предупреждения

Система должна показывать предупреждения до выдачи итогового проекта и внутри самого проекта.

Предупреждения нужны, если:

- нет подтвержденных норм потерь;
- нет проверенного выхода после фактического приготовления;
- не указан источник пищевой ценности;
- сроки хранения или температурные режимы не подтверждены;
- условия доставки или упаковки влияют на качество и безопасность;
- блюдо авторское или адаптированное и требует производственной проработки;
- данные нужны для документа, но пользователь их не предоставил.

## Граница "проект документа"

Проект документа появляется только после трех действий:

1. Пользователь подтвердил состав блюда.
2. Пользователь подтвердил общий технологический процесс.
3. Пользователь запросил формирование карты.

До этого момента ассистент работает с кулинарной гипотезой и расчетными заготовками. Это важно, чтобы пользователь не воспринимал раннее предложение как проверенную рецептуру.

## Что система должна явно показывать

- Тип проекта: ТК или ТТК.
- Статус: проект, не финально утвержденный документ.
- Какие данные пользователь подтвердил.
- Какие данные рассчитаны.
- Какие данные взяты из справочника.
- Какие данные требуют проверки.
- Какие данные требуют производственной проработки.
- Какие поля не заполнены.
- Какие поля нельзя считать подтвержденными без данных предприятия.

## Данные, которые часто требуют подтверждения

- Нормы потерь при холодной обработке.
- Нормы потерь при тепловой обработке.
- Выход готового блюда после фактической проработки.
- Условия хранения и реализации.
- Сроки годности.
- Температурные режимы.
- Пищевая ценность по конкретным продуктам.
- Требования к сырью и сопроводительным документам.
- Возможность доставки без потери качества и безопасности.

## Зона ответственности предприятия

После выдачи проекта начинается зона ответственности предприятия:

- производственная проработка;
- проверка фактического выхода и потерь;
- подтверждение сроков и условий хранения;
- проверка требований безопасности;
- проверка сырья и сопроводительных документов;
- финальное утверждение карты ответственным лицом;
- перенос данных в учетную систему.

Ассистент помогает подготовить структурированный проект. Он не заменяет технолога, лабораторные исследования, производственный контроль и внутреннее утверждение документа.
